в—Ҹ Plain English first before any code.

  Think of it like a bouncer at a door. Normally when your browser makes a
  request, it goes straight to the server and gets a real response back.
  page.route() lets you stand in the middle вҖ” intercept that request before it
  reaches the server, and hand back whatever fake response you want instead.

  Why is that useful? Imagine testing what happens when an API returns an error,
   or returns an empty list, or returns bad data. You don't want to depend on a
  real server to be in that broken state вҖ” you just fake it.

  The shape of it is:

  page.route(url to intercept, what to do instead)

  So for example вҖ” intercept any request to /posts, return a fake post you made
  up, then assert your page handled it correctly.

  One question before we write anything: in your SauceDemo tests, the browser
  never makes any API calls we can intercept. So we need a slightly different
  setup for this demo.
